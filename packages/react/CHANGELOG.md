# Changelog

## 0.1.0 - Unreleased

- Established the isolated KIN React integration package.
- Added direct official-package adapters for Sonner, NumberFlow, cmdk, React Virtuoso, dnd kit, input-otp, Liveline, and development-only Leva.
- Preserved Liveline's native Reduced Motion behavior and reserved `paused` for an actual paused-data state.
- Aligned Liveline's table fallback with its Unix-seconds time contract and caller-provided Canvas formatters.
- Documented the thin-adapter rule that preserves official behavior and motion while limiting KIN changes to public Token, theme, semantics, and verified exception boundaries.
- Preserved upstream motion and interaction engines while applying KIN semantics, Tokens, theme behavior, accessibility boundaries, and subpath exports.
- Added controlled Command Dialog focus return, dnd-kit activator focus, localized sortable announcements, first-value NumberFlow suppression, Liveline semantic fallback, bundle isolation, SSR smoke checks, and the Integration Lab acceptance suite.
- Added immutable-revision packing and consuming-product migration guidance without claiming a published package.
- Added safe-area-aware mobile Sonner offsets and runnable evidence for all six placements, LTR/RTL direction, success, undo, Promise updates, and recoverable failure while preserving the upstream motion engine.
- Preserved caller-owned OTP `aria-describedby` and `aria-invalid` values while merging KIN description and error references.
- Made Sonner localization extensible beyond English and Chinese, with explicit product-owned container and close labels taking precedence.
- Moved integration engines to exact optional peers and split adapter CSS so products can isolate installation, JavaScript, and stylesheet cost by subpath.
- Added clean cross-platform `dist` builds, package license inclusion, map-free tarballs, clean tarball installation, raw/gzip/CSS bundle reports, and dependency-boundary checks.
- Added isolated React 18.2 and React 19 server-render-to-`hydrateRoot` compatibility verification without mutating the repository lockfile.
