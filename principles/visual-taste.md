# Visual Taste Calibration

Taste is a sequence of decisions about hierarchy, density, rhythm and restraint. It cannot be outsourced to a component library or a numeric preset.

## KIN calibration

```yaml
design_variance: 3
motion_intensity: 4
visual_density: 8
decoration_level: 2
content_priority: 10
```

These numbers are review prompts, not runtime variables.

## Composition

- Establish one dominant work area.
- Let navigation and metadata recede after orientation.
- Align repeated values and actions precisely.
- Use asymmetry only when it clarifies task priority.
- Avoid marketing-page compositions inside an application.
- Do not default to three equal cards.

## Rhythm

- Space communicates relationships.
- Related controls sit closer than separate concepts.
- Dense rows may coexist with generous section breaks.
- Do not repeat a single gap value across an entire screen.
- Long Chinese text requires deliberate line length and line height.

## Hierarchy

Try these in order:

1. Content order
2. Alignment
3. Space
4. Size
5. Weight
6. Surface
7. Border
8. Color
9. Motion

If color or motion is doing the work of the first five, revisit the design.

## Density

High density means less repetition, not smaller everything.

- Keep primary rows around 36–44px where appropriate.
- Preserve 44px touch targets on touch interfaces.
- Collapse repeated labels through columns and grouping.
- Disclose rare properties progressively.
- Do not hide important status behind hover-only controls.

## Review prompts

- What is the first thing the user sees, and did it earn that attention?
- Which element can be removed without reducing understanding?
- Are repeated elements optically aligned?
- Are empty areas helping focus or revealing unfinished layout?
- Does the design still work in grayscale?
- Would an expert complete the task faster after one week of use?
- Does mobile preserve priority rather than merely stack desktop sections?
