# Product pattern references

These framework-free pages demonstrate how one KIN foundation produces different structures for different product tasks:

- [`information.html`](./information.html): reading, search, provenance, revision, and stable records.
- [`ecommerce.html`](./ecommerce.html): product comparison, money, inventory, channel state, and approval.
- [`canvas.html`](./canvas.html): tools, layers, geometry, units, selection, revision, and generated-change review.
- [`../workspace-reference/index.html`](../workspace-reference/index.html): entity, activity, evidence, and Inspector for intelligence work.

The pages are behavior and visual fixtures, not templates to copy unchanged. Use the matching document in [`patterns/`](../../patterns/) before adapting a reference.

Serve them with:

```bash
node scripts/serve-reference.mjs
```
