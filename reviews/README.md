# KIN review packets

Status: non-normative review material

This directory contains deterministic material prepared for named human review. A packet is evidence of what was presented for review; it is not proof that the review occurred and it is not a source of KIN requirements.

- [`agent-locales.md`](./agent-locales.md) joins each compact Agent rule with its exact normative source section, current English and Simplified Chinese copy, and candidate checksums.
- The locale packet is generated by `npm run agent:review:export` and checked by `npm run agent:review:check`.
- Do not edit generated packets. Correct the normative source or `distribution/locales/*.json`, regenerate the packet, and review the resulting diff.
- Human approval is recorded outside the packet. Any stored reviewer identifier must be a consented public handle or governed public attestation ID, never an email address or private identity. A separate reviewed change may then update checksum-bound locale attestations.

Review packets remain outside the GitHub Pages artifact allowlist unless a later, separately approved publication phase changes that boundary.
